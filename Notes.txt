## Step-by-Step Setup Instructions

### 1. Prerequisites

Install the following on your development machine:

- **Python 3.10+** - [Download](https://www.python.org/downloads/)
- **Node.js 18+** - [Download](https://nodejs.org/)
- **Git** - [Download](https://git-scm.com/)
- **VSCode** - [Download](https://code.visualstudio.com/)

**Recommended VSCode Extensions:**
- Python
- Django
- ES7+ React/Redux/React-Native snippets
- REST Client
- SQLite Viewer

### 2. Initial Project Setup

```bash
# Create project root directory
mkdir groci-smart
cd groci-smart

# Initialize git repository
git init
```

### 3. Backend (Django) Setup

#### 3.1 Create Django Project

```bash
# Create backend directory
mkdir backend
cd backend

# Create and activate virtual environment
python -m venv venv

# Activate virtual environment
# On Windows:
venv\Scripts\activate
# On Mac/Linux:
source venv/bin/activate

# Upgrade pip
pip install --upgrade pip

pip install azure-cognitiveservices-vision-computervision


# Install Django and dependencies
pip install django==4.2.7
pip install djangorestframework==3.14.0
pip install django-cors-headers==4.3.1
pip install pillow==10.1.0
pip install django-imagekit==5.0.0
pip install azure-cognitiveservices-vision-computervision==0.9.0
pip install msrest==0.7.1
pip install python-dotenv==1.0.0

# Create requirements.txt
pip freeze > requirements.txt

# Create Django project
django-admin startproject groci_backend .

# Create Django apps
python manage.py startapp accounts
python manage.py startapp receipts
python manage.py startapp products
python manage.py startapp shopping_lists
```

#### 3.2 Create Directory Structure

```bash
# Create service directories
mkdir -p receipts/services
mkdir -p products/services

# Create media directory
mkdir -p media/receipts

# Create static directory
mkdir static
```

#### 3.3 Create Environment File

Create `.env` file in `backend/` directory:

```env
# Django Settings
DJANGO_SECRET_KEY=your-secret-key-here-change-in-production
DEBUG=True

# Azure Computer Vision
AZURE_COMPUTER_VISION_KEY=your-azure-key-here
AZURE_COMPUTER_VISION_ENDPOINT=https://your-endpoint.cognitiveservices.azure.com/

# Database (SQLite for development)
# For production, you can add PostgreSQL/MySQL settings here
```

#### 3.4 Update Django Settings

Copy the `settings.py` content from the artifact provided earlier to:
`backend/groci_backend/settings.py`

#### 3.5 Create Models

Copy the model files from artifacts to respective locations:
- `backend/accounts/models.py`
- `backend/receipts/models.py`
- `backend/products/models.py`
- `backend/shopping_lists/models.py`

#### 3.6 Create Service Files

Create empty `__init__.py` files:

```bash
# In receipts/services/
touch receipts/services/__init__.py
touch receipts/services/ocr_service.py
touch receipts/services/receipt_parser.py

# In products/services/
touch products/services/__init__.py
touch products/services/price_comparison.py
```

Copy the `ocr_service.py` content from artifact to:
`backend/receipts/services/ocr_service.py`

#### 3.7 Create URLs Configuration

Copy URL configurations from artifacts:
- `backend/groci_backend/urls.py` (main URLs)
- `backend/accounts/urls.py`

Create empty URL files for other apps:

```bash
touch receipts/urls.py
touch products/urls.py
touch shopping_lists/urls.py
```

#### 3.8 Run Migrations

```bash
# Make migrations
python manage.py makemigrations

# Apply migrations
python manage.py migrate

# Create superuser (admin account)
python manage.py createsuperuser
```

#### 3.9 Test Backend Server

```bash
# Run development server
python manage.py runserver

# Visit http://127.0.0.1:8000/admin to test
```

### 7. Setting Up Azure Computer Vision

1. **Create Azure Account**: Visit [Azure Portal](https://portal.azure.com)
2. **Create Computer Vision Resource**:
   - Search for "Computer Vision"
   - Click "Create"
   - Select your subscription
   - Create or select resource group
   - Choose region (closest to you)
   - Select pricing tier (F0 for free tier)
3. **Get Credentials**:
   - After creation, go to resource
   - Navigate to "Keys and Endpoint"
   - Copy KEY 1 and ENDPOINT
   - Add to `.env` file

### 12. Troubleshooting

#### Backend Issues

**Issue**: `ModuleNotFoundError`
**Solution**: Ensure virtual environment is activated and dependencies installed

**Issue**: `CORS errors`
**Solution**: Check CORS settings in `settings.py` and ensure mobile app URL is in `CORS_ALLOWED_ORIGINS`

**Issue**: `Migration conflicts`
**Solution**: Delete migration files (except `__init__.py`) and db.sqlite3, then run migrations again

#### Frontend Issues

**Issue**: `Module not found`
**Solution**: Run `npm install` or `npx expo install package-name`

**Issue**: `Cannot connect to backend`
**Solution**: Use your computer's IP address instead of localhost in mobile app (e.g., `http://192.168.1.100:8000`)

**Issue**: `Expo not loading`
**Solution**: Clear cache with `npx expo start --clear`

### 13. Production Deployment (Future)

- **Backend**: Deploy to services like Heroku, Railway, DigitalOcean, or AWS
- **Database**: Migrate from SQLite to PostgreSQL
- **Media Files**: Use cloud storage (AWS S3, Cloudinary)
- **Mobile App**: Submit to Google Play Store and Apple App Store
- **Environment**: Use production environment variables
- **Security**: Update SECRET_KEY, disable DEBUG, configure ALLOWED_HOSTS

## Screen Breakdown

### 1. **Auth Stack** (No Tabs)
- **Login** - Email/username and password
- **Register** - Create new account
- **Forgot Password** - (Optional for future)

### 2. **Home Tab**
- **Dashboard** - Main screen with:
  - Monthly spending overview
  - Recent receipts (3-5)
  - Quick actions (Scan Receipt, New List)
  - Spending trends chart
  - Top stores

### 3. **Receipts Tab**
- **Receipts List** - All receipts with:
  - Filter by date, store, status
  - Search functionality
  - Pull to refresh
- **Receipt Detail** - View receipt:
  - Image preview
  - Items list
  - Total, tax, store info
  - Edit button
- **Upload Receipt** - Camera/gallery picker
- **Edit Receipt** - Update receipt info and items

### 4. **Shopping Lists Tab**
- **Lists Overview** - All shopping lists:
  - Active, completed, archived
  - Progress indicators
  - Create new list button
- **List Detail** - Individual list:
  - Items with check-off
  - Add/remove items
  - Price estimates
  - Total calculation
- **Create/Edit List** - Form to manage list
- **Price Comparison** - Compare prices across stores

### 5. **Profile Tab**
- **Profile Info** - User details
- **Settings** - App preferences
- **Preferred Stores** - Manage favorite stores
- **Change Password**
- **Logout**

## Navigation Patterns

### Primary Navigation
- **Bottom Tab Navigator** (4 tabs)
  - Home, Receipts, Lists, Profile

### Secondary Navigation
- **Stack Navigator** within each tab for sub-screens
- **Modal Screens** for:
  - Camera/Image Picker
  - Forms (Create/Edit)
  - Confirmation dialogs

### Gestures
- **Swipe back** - Native back gesture
- **Pull to refresh** - Refresh lists
- **Swipe to delete** - Delete list items

## Key Features of This Structure

### 1. **Expo Router File-based Routing**
- Uses folder structure for navigation
- `(auth)` and `(tabs)` are route groups
- Dynamic routes with `[id].tsx`
- Nested navigation with `_layout.tsx`

### 2. **Component Organization**
- **ui/** - Generic reusable UI components
- **feature folders** - Domain-specific components
- **common/** - Shared components across features

### 3. **Services Layer**
- Centralized API calls
- Easy to mock for testing
- Consistent error handling

### 4. **Context & Hooks**
- Global state management with Context API
- Custom hooks for data fetching
- Reusable logic extraction

### 5. **Type Safety**
- TypeScript types in dedicated folder
- API response types
- Component prop types

### 6. **Utilities**
- Formatters for dates, currency
- Validators for forms
- Helper functions

## File Naming Conventions

- **Components**: PascalCase (`ReceiptCard.tsx`)
- **Services**: camelCase (`authService.ts`)
- **Hooks**: camelCase with `use` prefix (`useAuth.ts`)
- **Types**: camelCase (`receipt.ts`)
- **Utils**: camelCase (`formatters.ts`)
- **Screens**: kebab-case for routes (`change-password.tsx`)
